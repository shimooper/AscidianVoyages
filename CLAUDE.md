## What This Project Does

ML research project predicting ship/zooplankton mortality along maritime routes based on environmental conditions (temperature, salinity, chlorophyll) sampled during voyages.

## Common Commands

### Run full analysis (local)
```bash
cd analyze_expirement_results
python main.py --outputs_dir_name outputs_test_local --cpus 1 --do_feature_selection False --run_configurations_in_parallel False --run_lstm_configurations_in_parallel False
```

### Run a single configuration
```bash
python analyze_expirement_results/run_analysis_of_config.py <path_to_config.csv>
```

### Score trained models on new routes
```bash
python analyze_expirement_results/score_routes/score_full_routes.py
```

### Run tests
```bash
python -m pytest tests/
```

### Prepare route data (one-time setup)
```bash
python prepare_routes/prepare_routes.py
```

## Architecture

Three main pipelines:

**`prepare_routes/`** — Processes raw shipping route coordinates, queries environmental conditions from NASA/CEDA datasets using KDTree spatial lookup, and produces `routes_with_conditions.csv`.

**`analyze_expirement_results/`** — The core ML pipeline:
- `main.py` generates all combinatorial configurations and dispatches them (locally or via SLURM)
- `run_analysis_of_config.py` runs one configuration end-to-end: preprocess → visualize → train models → aggregate results
- `preprocess.py` loads `Final_Data_Voyages.xlsx`, engineers temporal features (30-day windows, day-based intervals, acclimation days), and splits train/test
- `model.py` wraps sklearn classifiers (Decision Tree, Random Forest, KNN, Logistic Regression, Gradient Boosting, XGBoost, MLP) with GridSearchCV and optional RFECV/Optuna feature selection
- `model_lstm.py` + `train_lstm_cv.py` implement a PyTorch Lightning LSTM trained on (Temperature, Salinity, relative Time) sequences
- Both model types use 5-fold cross-validation; class imbalance handled via RandomUnderSampler
- `score_routes/` performs inference with trained `.pkl` (sklearn) or `.ckpt` (LSTM) checkpoints and aggregates route-level survival probabilities (min/max/mean/log-multiply)

**`prepare_environment_data/`** — Scripts to download and convert NASA/CEDA environmental datasets.

### Configuration System

`configuration.py` defines all experimental dimensions as lists that get crossed into combinatorial configs:
- `INTERVAL_LENGTH`: [1, 2, 3, 4] days of history to predict next day
- `METRIC_TO_CHOOSE_BEST_MODEL_HYPER_PARAMS`: 'mcc' or 'f1'
- `BALANCE_CLASSES_IN_TRAINING`, `STRATIFY_TRAIN_TEST_SPLIT`, `INCLUDE_CONTROL_ROUTES`, etc.

### DEBUG_MODE

Platform-detected in `configuration.py`:
- **Windows** → `DEBUG_MODE = True`: small datasets, 1 epoch, 10 Optuna trials, intervals [3,4] only
- **Linux** → `DEBUG_MODE = False`: full datasets, 15 epochs, 1000 Optuna trials, all intervals
